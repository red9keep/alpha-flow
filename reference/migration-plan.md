# AlphaFlow 블로그 디자인 및 레이아웃 이식 계획서

본 문서는 레퍼런스 폴더 `reference/alphaflow-export (1)` 내의 고품격 디자인, 레이아웃, 아이콘 에셋들을 실서비스 영역인 구글 블로그 소스(`blog_index.html`)와 깃허브 외부 관리 파일(`style.css`, `script.js`)로 완벽하게 이식하기 위한 상세 개발 계획서입니다.

---

## 1. 이식 전략 및 원칙 (Core Strategy)

1. **역할의 완전한 분리**
   - **`blog_index.html` (구글 블로그 테마 HTML):** 블로그의 고유 기능 구조와 레이아웃 뼈대만 유지합니다.
   - **`style.css` (깃허브 외부 CSS):** 레퍼런스의 `css/styles.css` 디자인 요소를 흡수하여 구글 블로그의 클래스들과 매핑하고, 전체 폰트, 컬러 톤앤매너(다크/화이트 모드 및 포인트 컬러), 레이아웃 스타일을 한 번에 제어합니다.
   - **`script.js` (깃허브 외부 JS):** 동적 레이아웃 생성, 사이드바 토글, 아이콘 인젝션, 대시보드 컴포넌트 추가 등 동적인 기능을 구현합니다.
2. **에셋 영구화 및 고속화**
   - 레퍼런스의 `images/` 폴더 내 로고(`logo.svg`) 및 핵심 아이콘(`zap.svg`, `bar-chart.svg` 등)을 깃허브 저장소로 업로드하고, **jsDelivr CDN**을 통해 블로그 내에서 무손실 고속 로딩이 가능하도록 절대 주소화합니다.

---

## 2. 세부 이식 계획 (Step-by-Step Plan)

### [Step 1] 정적 에셋(이미지/아이콘) 마이그레이션
- **목적:** 레퍼런스 이미지들을 깃허브 업로드용 구조로 전환
- **작업 내용:**
  - 프로젝트 루트에 `images/` 및 `images/icons/` 폴더 생성
  - `reference/alphaflow-export (1)/images/` 폴더 내의 로고 및 SVG 아이콘 파일들을 신규 생성한 폴더로 복사
  - 깃허브 업로드 시 사용할 절대 주소 확보:
    - 로고: `https://cdn.jsdelivr.net/gh/red9keep/alpha-flow@main/images/logo.svg`
    - 아이콘 예시: `https://cdn.jsdelivr.net/gh/red9keep/alpha-flow@main/images/icons/zap.svg`

### [Step 2] 스타일시트 이식 (`style.css` 개발)
- **목적:** 레퍼런스의 세련된 현대적 테마를 블로그 전체에 입힘
- **작업 내용:**
  - `reference/alphaflow-export (1)/css/styles.css`에서 정의된 레이아웃 규칙, CSS 변수(Variables: 컬러 톤, 여백, 둥글기 등), 미디어 쿼리(반응형 설정) 분석
  - 구글 블로그의 기본 스킨 스타일(CSS Reset)과 레퍼런스 스타일이 충돌하지 않도록 네임스페이스를 고려하여 `style.css` 작성
  - 주요 적용 요소:
    - 글꼴 (프리텐다드 베이스) 및 전체 배경 컬러 변경
    - 헤더/네비게이션 바의 미니멀 대시보드 스타일링
    - 본문 카드 섹션, 버튼, 툴팁, 사이드바 카드 디자인 반영

### [Step 3] 동적 레이아웃 및 아이콘 이식 (`script.js` 개발)
- **목적:** 대시보드 구조 및 동적 기능(토글 등) 및 SVG 아이콘 삽입을 구현
- **작업 내용:**
  - 구글 블로그 HTML을 복잡하게 뜯어고치지 않고, `script.js`에서 DOM 탐색기(`querySelector`)를 통해 동적으로 레퍼런스에 있는 세련된 헤더 로고, 퀵 네비게이션 메뉴, 사이드바 요소 등을 주입(Inject)
  - 레퍼런스 구조의 반응형 사이드바 토글 동작 등 자바스크립트 인터랙션 이식
  - 각 대시보드 카드 영역 및 위젯에 맞는 아이콘(SVG) 주입 스크립트 작성

### [Step 4] 구글 블로그 템플릿 최종 조율 (`blog_index.html`)
- **목적:** 외부 CSS/JS 가 최종적으로 올바른 블로그 영역에 결합되도록 함
- **작업 내용:**
  - 구글 블로그 본문 및 사이드바 위젯 영역(`b:section`, `b:widget`)이 외부 스타일시트 및 스크립트와 시각적으로 완벽하게 맞물리도록 필요한 영역에 간단한 컨테이너 클래스 지정
  - 구글 블로그의 기존 불필요한 구형 스타일 요소 비활성화 처리 (`b:css='false'` 등)

### [Step 5] 모바일 앱 스타일 하단 고정 네비게이션 바(Bottom Tab Bar) 추가
- **목적:** 모바일 기기 접속자를 위해 스마트폰 앱과 동일한 하단 고정 퀵 메뉴 제공
- **작업 내용:**
  - `script.js`를 통해 모바일 화면 최하단에 항상 고정되는 플로팅 네비게이션 바 레이아웃을 동적으로 인젝션
  - `reference` 내의 핵심 SVG 아이콘 에셋들을 각 메뉴 아이템에 배치:
    - **홈** (`home.svg`) -> 메인 홈 링크
    - **지표** (`bar-chart.svg`) -> 노동부 등 지표/대시보드 영역 바로가기
    - **추천** (`zap.svg`) -> 주요 추천 콘텐츠/피처 목록 바로가기
    - **알림** (`bell.svg`) -> 공지사항/공지 라벨 바로가기
    - **설정** (`settings.svg`) -> 사이트 소개/설정 메뉴 바로가기
  - `style.css`에 모바일 기기 화면 크기(`@media (max-width: 768px)`)에서만 활성화되는 반응형 하단 고정 CSS 작성
  - 세련된 앱 감성을 극대화하기 위해 반투명 블러 효과(`backdrop-filter: blur(10px)`) 및 탭 터치 인터랙션 디자인 반영

### [Step 6] 상단 롤링 뉴스 슬라이더 배너 구현
- **목적:** 블로그 최상단 핵심 영역에 실적 발표 일정, D-DAY 이슈 등 주요 공지 사항을 동적으로 슬라이딩하여 노출
- **작업 내용:**
  - `script.js` 내에 자동 롤링(4~5초 간격) 로직 및 터치/클릭 슬라이드 기능 구현
  - 빨간색 둥근 배경의 세련된 'D-DAY' 배지 디자인 마크업 및 CSS 설계
  - 우측 화살표(`chevron-right.svg`) 및 중앙 페이지 표시기(Dot Indicators) 인터랙션 추가

### [Step 7] 본문 멀티 탭 메뉴 및 아코디언 게시판 제어 시스템 구축
- **목적:** 복잡한 화면 스크롤 없이 클릭 한 번으로 메인/서브 카테고리를 필터링하고, 아코디언 토글 방식으로 깔끔하게 게시글을 조회하는 고성능 모바일 레이아웃 구현
- **작업 내용:**
  - **이중 중첩 탭(2-Tier Tabs) 개발:** 
    - 상위 메인 탭('알파종목', '최신뉴스', '보유자산') 제어 로직 설계
    - '최신뉴스' 활성화 시 하위에 세부 서브 탭('뉴스', '당잠사', '매드머니', '영상뉴스')이 나타나며, 타 메인 탭 클릭 시 서브 탭 바 전체가 숨겨지도록 연동
    - 선택된 서브 탭 아래에 굵은 액티브 밑줄(Underline Indicator) 인터랙션 추가
  - **아코디언 토글 게시판(Accordion Boards) 설계:**
    - 게시글 리스트 우측에 상세 더보기 토글 버튼 배치 (`chevron-down.svg` 에셋 활용)
    - 자바스크립트로 버튼 클릭 시 해당 아이템에 `open` 클래스를 토글하며, 상세 본문 영역을 부드러운 트랜지션 애니메이션(`transition: max-height 0.3s ease, opacity 0.3s ease`)으로 확장 및 축소
    - 열릴 때 우측 화살표 아이콘이 180도 회전(`transform: rotate(180deg)`)하여 상태를 직관적으로 나타내는 모션 구현

### [Step 8] 실시간 주식 티어 대시보드 시스템(stock.json) 연동
- **목적:** 수집 자동화 프로그램이 깃허브 저장소에 업로드하는 실시간 주식 데이터(`stock.json`)를 연동하여 화면에 동적으로 주식 티어표 렌더링
- **작업 내용:**
  - **데이터 흐름 아키텍처 설계:** 
    - 히스토리 추적용 날짜별 JSON 파일 생성과 별개로, 블로그에서 항상 가장 최근 데이터를 빠르고 캐시 없이 조회할 수 있도록 최상위에 고정된 `stock.json` 파일 생성 및 덮어쓰기 연동
    - 구글 블로그 `script.js`에서 깃허브 CDN 주소(`https://cdn.jsdelivr.net/gh/red9keep/alpha-flow@main/stock.json`)를 통해 데이터를 실시간 `fetch`
  - **데이터 바인딩 및 동적 UI 렌더링:**
    - JSON 내부의 티어 그룹(1티어, 2티어 등) 단위로 영역을 감싸고, 하위의 개별 종목 데이터들을 루프를 돌며 동적 카드 형태로 생성
    - 등락 정보(상승/하락)에 따라 텍스트 컬러(빨간색/파란색) 및 심볼 아이콘 실시간 변경 처리
    - 종목의 특정 상태 태그(예: "시간 손절", "수집 완료")에 부합하는 개별 맞춤형 세련된 아웃라인 배지 스타일 렌더링
    - 종목별 리스트 우측 아코디언 토글 버튼(`chevron-down.svg`) 연동을 통해 클릭 시 종목별 추가 기술 정보나 세부 사항 영역 활성화

---

## 3. 이식 후 예상되는 기대 효과

- **모던 대시보드화:** 기존 구글 블로그의 정형화된 느낌에서 완전히 벗어나, 핀테크/SaaS 서비스 느낌의 세련된 대시보드 웹 애플리케이션 형태의 블로그로 재탄생합니다.
- **간편한 업데이트:** 향후 텍스트나 로고, 메뉴 구성, 서체 굵기 등을 바꿀 때 구글 블로그에 직접 접속할 필요 없이 **VS Code에서 `style.css`와 `script.js`만 고쳐서 push**하면 즉각 적용됩니다.
- **통합 관리:** 모든 에셋(이미지, CSS, JS)이 단일 저장소(`alpha-flow`) 안에서 버전 관리되므로 안정성이 매우 높아집니다.
